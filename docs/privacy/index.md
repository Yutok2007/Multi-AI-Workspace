Multi-AI Workspace Privacy Policy

Last updated: July 23, 2026

1. Information Handled by the Extension

Multi-AI Workspace may process and store extension settings, interface preferences, prompt templates, unsent drafts, conversation URLs and titles, timeline metadata, pin and highlight positions, export history, and user-configured AI provider information on the user’s device.

To provide chat enhancement features, the extension reads visible chat composers, prompts, and conversation content on supported AI websites opened by the user.

2. How Information Is Used

Information is used only to provide the extension’s disclosed functionality, including prompt management and rewriting, draft restoration, quoting and highlighting, pinning, timelines, conversation branching, chat summarization, exporting, and Mermaid diagram rendering.

The extension does not use user information for advertising, user profiling, credit assessment, or purposes unrelated to its disclosed single purpose.

3. Local Storage and Retention

Most extension data is stored in the browser’s local or session storage.

AI provider API keys are stored in browser session storage by default. If a user chooses persistent storage, the API key is encrypted locally using AES-GCM and a password supplied by the user.

Locally stored information remains until the user deletes or resets it, clears the extension’s browser data, or uninstalls the extension.

4. Third-Party AI Providers

Prompt or conversation text is sent to a third-party AI provider only when the user explicitly configures that provider and invokes a provider-based feature, such as Prompt Rewrite or Summarize Chat.

Only information necessary to complete the requested operation is sent. The information is sent directly from the extension to the provider selected by the user and is not routed through a server operated by the developer.

The selected provider processes information according to its own privacy policy and terms. Users should review those policies before configuring a provider.

5. Data Sharing and Sale

The developer does not sell user data.

User data is not transferred to third parties except when necessary to complete a feature explicitly requested by the user, comply with applicable law, or protect the security of the extension and its users.

The developer and its personnel do not manually read users’ prompts, conversations, or API keys.

6. Extension Permissions

The storage permission is used to save extension settings and local workspace data.

The optional notifications permission is used only to send answer-completion notifications after the user enables that feature.

Host permissions are used to provide visible page enhancements on supported AI chat websites and to communicate with AI provider endpoints explicitly configured by the user.

7. Security

The extension does not download or execute remote code.

Communications with remote AI providers use HTTPS. The only exception is a localhost service explicitly configured by the user, such as a locally installed Ollama endpoint.

The extension does not read AI platform passwords or authentication cookies.

8. User Controls

Users can disable provider features, notifications, and page enhancements from the extension settings.

Users can delete or reset locally stored extension information. Users may also avoid configuring an external AI provider.

Conversation context is not automatically included in Prompt Rewrite requests unless the user enables the relevant setting or explicitly requests a context-dependent operation.

9. Chrome Web Store User Data Policy

Multi-AI Workspace’s use and transfer of user information complies with the Chrome Web Store User Data Policy, including the Limited Use requirements.

10. Changes to This Privacy Policy

This policy may be updated when the extension’s functionality or data practices change. The latest version and its effective date will be published on this page.
