# GPT-Voyager public-PRD gap matrix

This matrix compares Multi-AI Workspace with behaviors described in the public GPT-Voyager PRD. It is a product-planning aid, not a source-code comparison. GPT-Voyager source, styles, selectors, assets, and translations were not used.

| Publicly described behavior                                           | Multi-AI Workspace status                                                 | Notes                                                                                                                                                                                      |
| --------------------------------------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Prompt CRUD, tags, favorites, import/export, and variables            | Implemented                                                               | Prompt-folder management is not yet exposed; clipboard variables require a user gesture                                                                                                    |
| Selected-text rewrite, quote reply, and highlighting                  | Implemented                                                               | Rewrite works without page binding; quote/highlight require the corresponding validated capability; highlight records store no text                                                        |
| Provider-backed chat summarization                                    | Implemented for safely detected visible messages                          | Sends only user/assistant plain text after explicit confirmation, excludes tool/system/HTML content, and keeps the structured result in memory                                             |
| In-conversation selected-range pins                                   | Implemented for automatically detected or manually bound visible messages | Pins are per conversation, expose separate dots plus previous/next navigation, store offsets and a verification hash without selected text, and remain local-only                          |
| Conversation branching from a selected message                        | Implemented across all built-in supported sites                           | Nearby buttons create a bounded text-only preview, copy or open a new chat, then use an expiring session-only explicit-insert handoff; nothing is auto-sent                                |
| Markdown, JSON, and self-contained sanitized HTML conversation export | Implemented for explicitly bound visible messages                         | PDF and bulk-account export are not implemented                                                                                                                                            |
| Timeline navigation, notes, hierarchy, and selected-node export       | Implemented for automatically detected or manually bound visible messages | Includes a deduplicated dot rail with hover/focus summaries plus active-node tracking, filters, persisted hierarchy/collapse/note metadata, and selected-node export                       |
| Mermaid diagram rendering/export                                      | Implemented for explicit Mermaid blocks in safely detected messages       | Renderer is locally bundled and loaded on demand; strict mode, source limits, timeouts, SVG sanitization, source preservation, code/diagram switching, and SVG/PNG export are enforced     |
| Formula LaTeX/MathML collection                                       | Implemented for verified rendered formula sources                         | Extracts KaTeX annotations, MathJax source nodes, MathML, and explicit source attributes; render-only formulas offer compatible text and clearly labeled local approximate conversion      |
| Seasonal visual effects                                               | Implemented                                                               | The popup directly switches Off/Snow/Sakura/Rain without binding; one pointer-transparent Canvas pauses in hidden tabs or reduced-motion mode and uses bounded responsive particle density |
| IDE context export                                                    | Not implemented                                                           | No local port scanning, native host, file access, or shell execution is authorized                                                                                                         |
| Automatic account isolation                                           | Partial                                                                   | Records are account-scoped, but adapters currently fall back to `anonymous` until a stable visible account scope can be verified                                                           |
| Whole-account automatic import                                        | Intentionally not implemented                                             | The extension does not use private APIs, cookies, tokens, or automated login                                                                                                               |

The next clean-room priority is IDE context export. It must be implemented through this repository's capability contracts and tested without relying on private platform APIs.
