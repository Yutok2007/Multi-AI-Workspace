# Manual Test Matrix

Automated checks cover local adapters, Provider request/response validation, storage/migrations and secret cleanup, URL sanitization, shortcut behavior, SPA route detection, production popup/options UI, bilingual switching, selected-text highlighting/quote actions, host-DOM preservation, and page-level Prompt navigation. Real signed-in sites still require user-run manual checks because the project does not receive credentials and never automates login.

## Per-platform procedure

Repeat on ChatGPT, Claude, Gemini, DeepSeek, Grok, and Kimi:

1. Load the appropriate unpacked build and sign in manually.
2. Confirm the host page remains usable before and after opening the `MW` Shadow DOM panel.
3. Finish popup privacy onboarding and confirm the popup opens compact settings directly, with no platform-readiness or binding cards. Toggle one setting and confirm it persists after reopening; switch **中文 / EN** and confirm popup, full settings, and page panel update.
4. Confirm the composer connects automatically when there is one high-confidence editable candidate. If it does not, use **Connect** to correct only the missing composer/send control and confirm read/write preserves host input events.
5. Confirm visible user Prompts are detected automatically and AI responses are not listed as Prompts. If detection is uncertain, bind representative user and AI messages manually; the extension must reject identical selectors for both roles.
6. Confirm drafts restore only into an empty composer and clear only after a confirmed send leaves it empty.
   Navigate to another conversation through the site's SPA UI and confirm the old conversation's draft or completion state does not leak into the new route.
   Under **Settings → Experimental**, enter an exact visible default model for the platform. In **Connect**, bind the visible model control. Open a genuinely new empty chat and confirm the configured model is visibly selected; change the setting and confirm the next new chat uses the new value. Return to an existing conversation and confirm its model is unchanged. Enter an unavailable model and confirm the platform default remains selected with no inferred fallback.
7. Configure a test Provider, rewrite a harmless prompt, and confirm the composer changes only after Replace/Insert/Append.
8. Test Prompt Manager variables, including clipboard only after clicking Insert.
9. Select visible page text and confirm **Improve Prompt** appears without message binding. Activate **Pin a spot**, click a text-free page position, and confirm one separate pin dot appears. Verify the hover list, previous/next controls, direct jump to the clicked height, deletion/undo, compatible reload restoration, and isolation after switching conversations. After composer and message connection, confirm **Quote Reply** inserts a quoted block without sending. Apply yellow twice and confirm only one layer remains; apply blue to a partial overlap and confirm blue replaces yellow only in that overlap; remove the selected highlight and confirm no yellow layer reappears underneath. Reload to confirm restoration and verify the host message HTML is unchanged.
   For formulas with exact KaTeX/MathJax/MathML source, confirm only available exact copy formats appear. For a render-only formula, confirm the dialog shows **Copy compatible formula** plus clearly labeled approximate LaTeX/Word/Notion actions and no disabled-format row.
   Generate one response containing two fenced `mermaid` blocks. Confirm both render inline only after their source settles, fit the response width, and change theme with the page. Click a diagram to open the existing source/export dialog. Test malformed Mermaid and confirm the original code remains visible with a local error and the rest of the answer remains usable.
10. Confirm every user Prompt has exactly one dot in the right-side navigator, including when the page uses nested role elements. Confirm the summary card is hidden by default, appears on hover or keyboard focus, and hides after leaving. Click first/middle/last dots and summaries to verify direct scrolling, and confirm the navigator does not block the open MW panel. In the panel timeline, verify the current node follows page scrolling; test search and Prompt/note filters; add and reload a node note; indent/outdent; collapse/expand descendants; select individual and visible nodes; and export the selection as Markdown/JSON/HTML. Confirm there is no message-favorite category, star action, or starred filter. For each visible user/assistant message, click its nearby branch button and verify the preview stops at that message, omits tool/system/attachment content, displays the sanitized source URL, and warns when bounded. Test Copy Context and Open New Chat; in the new page, confirm context appears only in the extension handoff banner, requires an explicit Insert click, preserves existing composer content at the cursor, never clicks Send, and disappears after insert/discard or 15 minutes. On every supported platform, confirm no extension-owned folder sidebar or folder control is injected. Verify only timeline, branching, and export request readable messages.
    Open **Conversation → Summarize the chat**. With no Provider, confirm the configuration action is shown. With a Provider, choose the output language, confirm the disclosure names the visible message/character count, cancel once to verify no request is made, then approve and verify the structured summary, copy action, loading lock, and error recovery. Confirm system/tool messages and HTML are never included in the request and the result disappears after a reload.
11. While an answer is generating, bind its visible Stop/progress indicator if the page does not expose `aria-busy`. Enable notifications and confirm there is no notification at the first token or during streaming. Confirm exactly one content-free notification after the indicator ends and the final message settles. Repeat with Stop/cancel, a visible error, and a disconnected request; none may send an “answer complete” notification. Remove notification permission and confirm answering still works normally.
12. Export raw data after at least one Provider-key migration and confirm neither current nor migration-backup Provider secrets appear in the JSON.
13. Click the extension icon and switch between Snow, Sakura, Rain, and Off in the Background Effects quick control; repeat under **Settings → Experimental**. Confirm both controls stay synchronized and each mode updates immediately without binding; does not block clicking, selecting, typing, or scrolling; survives a same-tab SPA conversation change; resizes with the viewport; stops in a background tab; and remains disabled when the operating system requests reduced motion.
14. Upgrade from a build that authorized a custom website and confirm its legacy dynamic script, record, and unused origin permission are removed. Reset data and confirm optional Provider origins, notifications, local records, background/session secrets, and encrypted Provider secrets are removed.
15. Export diagnostics and confirm it contains no conversation content, email, cookies, tokens, account identifiers, or Provider keys.
16. Open the page panel **Status** tab and run **Compatibility self-check**. Confirm Pin, Timeline, Branch, Quote Reply, and Export each show Available, Built in, Disabled, Retrying, or Unavailable with a reason. Switch conversations through the platform UI, then replace or locally refresh the conversation area without changing the URL; confirm the monitor reconnects and the features work without another Connect action. Temporarily hide/remove the composer and messages in developer tools and confirm dependent features enter Safe fallback without blocking native typing, clicking, or navigation, then restore the elements and confirm recovery. Export the safe report and verify it contains no page URL, selector, account identifier, error message, prompt, or response text.
17. Open the **MW** page shortcut menu on each supported platform. Confirm it contains only actions proven usable on the current page: Prompt requires the feature switch plus a validated writable composer, Pin requires a valid pin scope, Branch requires the branch capability plus readable messages, Export requires the export switch plus readable messages, and Visual Effects is hidden under reduced motion. Search for a saved Prompt, insert it, verify variables expand and text is written at the cursor without sending, and confirm the menu closes after insertion. Verify Pin enters exact-position mode, Branch exposes only the current conversation's message positions, Export uses the configured format, and every visual effect updates immediately. Switch conversations through native SPA navigation and repeat without rebinding. Confirm the full workspace has no duplicate Prompt tab, **Open full workspace** remains available for detailed status/settings, and clicking outside or pressing Escape closes only the shortcut menu.
